# 회의실 예약 시스템 (Meeting Room Booking)

구글 캘린더 UI/UX를 활용한 **회사 내부 회의실 예약 시스템**입니다.
**계정 가입이 필요 없고**, 예약 시 이름만 기재하면 캘린더에 바로 표시됩니다.

## 주요 기능

- 📅 **구글 캘린더 스타일 주간 뷰** — 요일 컬럼 + 시간대, 오늘 강조, 현재 시각 빨간선
- 🖱️ **드래그로 시간대 선택 → 예약 생성** — 원하는 범위를 끌어서 지정하면 모달이 열리고, 모달이 떠 있는 동안 선택 영역이 유지 표시
- ✏️ **예약 수정** — 이벤트 클릭 → 팝오버에서 수정/삭제
- 🗂️ **히스토리 페이지** (`/history`) — 전체/예정/지난 필터, 제목·예약자·메모 검색, 날짜별 그룹
- 👤 **계정 불필요** — 이름을 한 번 기재하면 브라우저에 저장되어 다음 예약 시 자동 입력
- 🎨 **예약자별 고정 색상** — 새 이름 등록 시 구분 잘 되는 16색 팔레트에서 랜덤 배정 후 `data/colors.json`에 영구 저장 (팔레트 소진 시 골든앵글로 자동 생성), 사이드바에 색상 범례 표시
- ⛔ **이중 예약 방지** — 같은 회의실·같은 시간이 겹치면 예약 거부 (서버 검증)
- 📱 모바일 반응형 — 모바일에서는 플로팅 + 버튼으로 예약 생성

## 기술 스택

- **Next.js 16** (App Router) + **React 19** + **TypeScript**
- **Tailwind CSS v4**
- 저장소: `data/reservations.json` **파일 기반** (별도 DB·계정 불필요)

## 실행 방법

```bash
npm install
npm run dev       # http://localhost:3000
```

프로덕션:

```bash
npm run build
npm run start
```

## 운영 정책 (기본값)

- 예약 가능 시간: **07:00 ~ 22:00**, **30분 단위** (`lib/validate.ts`)
- 회의실 목록: `lib/rooms.ts` 의 `ROOMS` 배열 편집 (이름·인원·위치·색상)
- 입력 제한: 제목 100자 · 예약자 30자 · 메모 300자 (서버에서도 검증)

## 데이터 저장소

`lib/store.ts` 가 환경에 따라 자동으로 백엔드를 선택합니다:

| 환경 | 백엔드 | 조건 |
|---|---|---|
| **프로덕션 (Vercel)** | Upstash Redis | `UPSTASH_REDIS_REST_URL`(또는 `KV_REST_API_URL`) 환경변수 존재 시 — Vercel 마켓플레이스 연동이 자동 주입 |
| **로컬 개발** | JSON 파일 (`data/`) | Redis 환경변수가 없을 때. 경로는 `DATA_DIR` 로 변경 가능 |

파일 백엔드 안전장치:
- **원자적 쓰기** — 임시 파일에 쓴 뒤 rename하므로 쓰기 도중 중단돼도 원본이 잘리지 않음
- **파손 백업** — JSON 파싱 실패 시 `*.corrupt-<timestamp>.json` 으로 백업 후 빈 상태로 시작 (조용한 유실 방지)
- **쓰기 직렬화** — 프로세스 내 큐로 동시 요청 race 방지

> 참고: 쓰기 직렬화는 프로세스 단위입니다. 소규모 내부용 트래픽 전제이며,
> 대규모 동시 쓰기가 필요해지면 Redis 트랜잭션(WATCH/MULTI) 또는 DB로 전환하세요.

## API

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/reservations` | 전체 예약 조회 |
| POST | `/api/reservations` | 예약 생성 (검증 실패 400, 겹침 409) |
| PATCH | `/api/reservations/:id` | 예약 수정 (없으면 404, 겹침 409) |
| DELETE | `/api/reservations/:id` | 예약 삭제 (없으면 404) |
